【项目概述】
--------------------------------------------------------------------------------
本项目实现了 accounts 微服务通过 Feign Client 调用 cards 和 loans 微服务的

3. 微服务客户端服务层
--------------------------------------------------------------------------------
文件：accounts/src/main/java/org/yiqixue/accounts/service/client/MicroserviceClientService.java

提供三个核心方法：

@Service
@AllArgsConstructor
public class MicroserviceClientService {
    
    private final CardsFeignClient cardsFeignClient;
    private final LoansFeignClient loansFeignClient;
    
    // 查询卡片详情
    public Object getCardDetails(String mobileNumber) {
        return cardsFeignClient.fetchCardDetails(mobileNumber);
    }
    
    // 查询贷款详情
    public Object getLoanDetails(String mobileNumber) {
        return loansFeignClient.fetchLoanDetails(mobileNumber);
    }
    
    // 查询完整客户信息（含容错处理）
    public CustomerFullDetails getCustomerFullDetails(String mobileNumber) {
        Object cardDetails = null;
        Object loanDetails = null;
        
        try {
            cardDetails = cardsFeignClient.fetchCardDetails(mobileNumber);
        } catch (Exception e) {
            System.err.println("Failed to fetch card details: " + e.getMessage());
        }
        
        try {
            loanDetails = loansFeignClient.fetchLoanDetails(mobileNumber);
        } catch (Exception e) {
            System.err.println("Failed to fetch loan details: " + e.getMessage());
        }
        
        return new CustomerFullDetails(cardDetails, loanDetails);
    }
}

设计亮点：
  - 使用构造函数注入依赖（@AllArgsConstructor）
  - 提供容错机制，单个微服务调用失败不影响整体流程
  - 内部类 CustomerFullDetails 封装返回结果


4. 客户服务实现（核心业务逻辑）
--------------------------------------------------------------------------------
文件：accounts/src/main/java/org/yiqixue/accounts/service/impl/CustomerServiceImpl.java

@Service
@AllArgsConstructor
public class CustomerServiceImpl implements ICustomerService {

    @Autowired
    CardsFeignClient cardsFeignClient;
    
    @Autowired
    LoansFeignClient loansFeignClient;
    
    @Override
    public CustomerDetailsDto fetchCustomerDetails(String mobileNumber) {
        // 1. 从本地数据库查询客户信息
        Customer customer = customerRepository.findByMobileNumber(mobileNumber)
            .orElseThrow(() -> new ResourceNotFoundException("Customer", "mobileNumber", mobileNumber));
        
        // 2. 从本地数据库查询账户信息
        Accounts accounts = accountsRepository.findByCustomerId(customer.getCustomerId())
            .orElseThrow(() -> new ResourceNotFoundException("Account", "customerId", customer.getCustomerId().toString()));
        
        // 3. 组装基础客户信息
        CustomerDetailsDto customerDetailsDto = CustomerMapper.mapToCustomerDetailsDto(customer, new CustomerDetailsDto());
        customerDetailsDto.setAccountsDto(AccountsMapper.mapToAccountsDto(accounts, new AccountsDto()));
        
        // 4. 调用 loans 微服务获取贷款信息
        ResponseEntity<Object> loansResponse = loansFeignClient.fetchLoanDetails(mobileNumber);
        if (loansResponse.getBody() != null) {
            ObjectMapper objectMapper = new ObjectMapper();
            LoansDto loansDto = objectMapper.convertValue(loansResponse.getBody(), LoansDto.class);
            customerDetailsDto.setLoansDto(loansDto);
        }
        
        // 5. 调用 cards 微服务获取卡片信息
        ResponseEntity<Object> cardsResponse = cardsFeignClient.fetchCardDetails(mobileNumber);
        if (cardsResponse.getBody() != null) {
            ObjectMapper objectMapper = new ObjectMapper();
            CardsDto cardsDto = objectMapper.convertValue(cardsResponse.getBody(), CardsDto.class);
            customerDetailsDto.setCardsDto(cardsDto);
        }
        
        return customerDetailsDto;
    }
}

关键实现细节：
  (1) 数据转换：使用 ObjectMapper.convertValue() 将 Feign 返回的 LinkedHashMap 
      转换为具体的 DTO 对象
  (2) 空值检查：在转换前检查 getBody() != null，避免空指针异常
  (3) 异常处理：如果微服务调用失败，抛出 ResourceNotFoundException

【关键技术要点】
================================================================================

1. Feign Client 返回类型选择
--------------------------------------------------------------------------------
问题：Feign Client 应该返回什么类型？

解决方案：使用 ResponseEntity<Object>
  - 原因：不同微服务返回的 DTO 结构不同，使用 Object 可以保持通用性
  - 转换：在 Service 层使用 ObjectMapper.convertValue() 转换为具体 DTO


2. 数据转换策略
--------------------------------------------------------------------------------
// Feign 返回的是 LinkedHashMap
ResponseEntity<Object> response = cardsFeignClient.fetchCardDetails(mobileNumber);

// 使用 ObjectMapper 转换为具体 DTO
ObjectMapper objectMapper = new ObjectMapper();
CardsDto cardsDto = objectMapper.convertValue(response.getBody(), CardsDto.class);

为什么需要转换？
  - Feign 默认将 JSON 反序列化为 LinkedHashMap
  - 需要转换为强类型的 DTO 才能进行类型安全的操作


3. 容错处理
--------------------------------------------------------------------------------
在 MicroserviceClientService.getCustomerFullDetails() 中使用 try-catch：

try {
    cardDetails = cardsFeignClient.fetchCardDetails(mobileNumber);
} catch (Exception e) {
    System.err.println("Failed to fetch card details: " + e.getMessage());
}

好处：
  - 即使某个微服务不可用，仍能返回部分数据
  - 提高系统的可用性和健壮性


4. 服务发现集成
--------------------------------------------------------------------------------
通过 Eureka 实现服务发现：
  - Feign Client 使用服务名称（name = "cards"）而非硬编码 URL
  - 支持负载均衡和故障转移
  - 需要在 application.yml 中配置 Eureka 服务器地址

【最佳实践总结】
================================================================================

推荐做法：
--------------------------------------------------------------------------------
1. Feign Client 放在 common 模块
   - 便于多个微服务复用
   - 统一管理接口定义

2. 使用 ResponseEntity<Object> 作为返回类型
   - 保持灵活性
   - 在 Service 层进行类型转换

3. 添加容错处理
   - 避免因单个微服务故障导致整体失败
   - 记录错误日志便于排查

4. DTO 结构保持一致
   - accounts 中的 CardsDto/LoansDto 与对应微服务完全一致
   - 确保数据转换的正确性

5. 参数验证
   - 使用 @Pattern 验证手机号格式
   - 在 Controller 层进行输入校验


注意事项：
--------------------------------------------------------------------------------
1. 服务启动顺序
   - 先启动 eurekaserver
   - 再启动 cards 和 loans
   - 最后启动 accounts

2. Eureka 注册
   - 确保所有微服务都正确注册到 Eureka
   - 检查 application.yml 中的 Eureka 配置

3. 网络超时
   - Feign 默认超时时间较短
   - 生产环境需要配置合理的超时参数

4. 循环依赖
   - 避免微服务之间的循环调用
   - 使用异步消息队列解耦复杂依赖
